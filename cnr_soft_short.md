# Шпаргалка: Soft Skills Interview

## 1. Product Sense — продукт с плохим UX
- Важные кнопки скрыты в меню, трачу время на поиск
- Решение: кастомизируемый home screen с частыми действиями

---

## 2. Сложная система
- Устаревший README, непонятно как запустить проект
- Внутренние действия (roles, feature flags) не задокументированы
- Решение: актуальная документация + автоматизация скриптами

---

## 3. Плохой код
- Django views: всё в одном месте — валидация, логика, БД
- Решение: service layer, serializers для валидации, managers для данных

---

## 4. Любимый проект
- Рефакторинг API — упростил структуру, ускорил response time
- Автоматизация deployment — меньше ручной работы, меньше ошибок
- Background jobs для отчётов — async обработка, лучший UX

---

## 5. Framework (Django)
- **Одно предложение**: batteries-included, ORM/admin/auth из коробки
- **Ключевые концепции**: MTV, ORM, Apps, Admin, Middleware
- **Сложности**: N+1 queries, сложные запросы в ORM
- **Улучшения**: хотелось бы DI как в FastAPI

---

## 6. Как изменился взгляд на инженерию
- Раньше: "идеальный код"
- Сейчас: deliver value, iterate fast, simplicity first
- Тесты = safety net для уверенных изменений

---

## 7. Критический фидбек
- "Двигаешься слишком быстро, упускаешь детали"
- Добавил extra self-review перед сабмитом PR

---

## 8. Конфликт с коллегой
- Коллега смешивал refactoring с основной задачей в PR
- Поговорил напрямую, попросил делать focused PRs
- Решилось без эскалации

---

## 9. Выбор новой БД
- Redis — кэширование, rate limiting (atomic ops)
- Elasticsearch — full-text search с фильтрами
- Read replica — разделение read/write нагрузки

---

## 10. Менторинг — что объяснить сразу
- **Clarity first** — уточняй до начала работы
- **Small PRs** — легче review, test, rollback
- **Simplicity** — простое решение до абстракций
- **Communicate early** — blockers сразу, не в конце

---

## 11. Инсайт из книги/подкаста
- "Thinking, Fast and Slow" — два режима мышления
- В инженерии легко полагаться на интуицию
- Замедлиться и подумать = меньше ошибок
